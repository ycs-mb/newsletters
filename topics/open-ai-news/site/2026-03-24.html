Tip of the Day

Use gpt-5.4-nano as your first-pass model and escalate to gpt-5.4 only on failure.

Most agentic subtasks — classification, routing, data extraction, well-defined coding subagents — don't require frontier-level reasoning. gpt-5.4-nano at $0.20/1M input tokens is 15–20× cheaper than the flagship. The pattern:

  1. Have the nano/mini model return a confidence score in structured output (JSON mode).
  2. If confidence ≥ threshold, accept the result. If below, escalate to gpt-5.4.
  3. Only route ambiguous or multi-step reasoning tasks to the flagship from the start.
from openai import OpenAI
client = OpenAI()

def run_with_escalation(prompt: str, threshold: float = 0.85) -> str:
    # First pass: nano model with confidence signal
    resp = client.chat.completions.create(
        model="gpt-5.4-nano",
        messages=[{"role": "user", "content": prompt}],
        response_format={"type": "json_object"}
    )
    data = json.loads(resp.choices[0].message.content)

    if data.get("confidence", 0) >= threshold:
        return data["answer"]

    # Escalate to flagship only when needed
    resp2 = client.chat.completions.create(
        model="gpt-5.4",
        messages=[{"role": "user", "content": prompt}]
    )
    return resp2.choices[0].message.content

Teams running this pattern at scale report 60–80% cost reduction with minimal quality loss on end-to-end task accuracy. OpenAI's production best practices guide explicitly recommends: "Use the smallest model that meets quality requirements — test with cheaper models first."
