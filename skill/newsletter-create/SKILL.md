# Newsletter Create

Generate a complete newsletter automation prompt for any topic, ready to run with Claude Code.

## Usage

```
/newsletter-create <topic-slug> [description]
```

Examples:
```
/newsletter-create rust-lang
/newsletter-create kubernetes "Cloud native and K8s ecosystem"
/newsletter-create indie-hacking "Bootstrapped startups, revenue milestones, launches"
```

## What This Skill Does

1. Asks clarifying questions about the topic (if not obvious from args)
2. Creates `~/newsletters/topics/<topic-slug>/` with `prompt.md` and `site/template.html`
3. Copies shared style asset into the topic's site folder
4. Adds an entry to `~/newsletters/topics.toml`
5. Produces a prompt that follows the exact same structure as the Claude Digest

## Step-by-Step Process

### Step 1: Parse Input

Extract from args:
- `topic-slug`: kebab-case identifier (e.g. `rust-lang`, `ai-safety`, `indie-hacking`)
- `description`: optional one-liner — if missing, ask the user

### Step 2: Gather Topic Details

Ask the user (skip if already clear from args/description):

1. **Newsletter name** — e.g. "Rust Weekly Digest", "K8s Signal"
2. **Eyebrow label** — subtitle shown above the title, e.g. "Daily Intelligence Brief"
3. **Accent color** — one of: `terracotta`, `sage`, `prussian`, `gold`
4. **Signal label** — the rating label shown in the header, e.g. "Signal", "Escalation", "Activity"
5. **Primary sources** — official blogs, docs, org sites to search
6. **GitHub focus** — what repo categories to look for (frameworks, tools, libraries)
7. **Community sources** — subreddits, Discord servers, X/Twitter accounts, HuggingFace tags
8. **Section customization** — keep the 5-section structure but adapt titles:
   - Section 01: Official / Core updates (releases, changelogs, RFCs)
   - Section 02: Highlights (major announcements, breaking changes)
   - Section 03: GitHub Picks (repos, tools, libraries)
   - Section 04: Community & Research (Reddit, Twitter, papers, forums)
   - Section 05: Tip of the Day (actionable technique)

### Step 3: Create Directory Structure

```
~/newsletters/topics/<topic-slug>/
  prompt.md          # Generated prompt
  site/
    index.html       # Will be generated on each run
    style.css        # Copied from ~/newsletters/shared/assets/style.css
    template.html    # Topic-specific template (generated by this skill)
```

Copy the shared stylesheet and create the site directory:
```bash
mkdir -p ~/newsletters/topics/<topic-slug>/site
cp ~/newsletters/shared/assets/style.css ~/newsletters/topics/<topic-slug>/site/
```

Then generate a `template.html` for the topic by copying an existing topic's template and updating the accent color and signal label. Use the claude-digest template as the canonical reference:
```bash
cp ~/newsletters/topics/claude-digest/site/template.html ~/newsletters/topics/<topic-slug>/site/template.html
```
Edit the copied template to update:
- `data-accent="terracotta"` → `data-accent="{ACCENT}"`
- The signal label text if it differs from "Signal"

### Step 4: Add to topics.toml

Append to `~/newsletters/topics.toml`:

```toml
[{TOPIC_SLUG}]
name = "{TOPIC_NAME}"
description = "{TOPIC_DESCRIPTION}"
accent = "{ACCENT}"
signal_label = "{SIGNAL_LABEL}"
folder = "topics/{TOPIC_SLUG}"
eyebrow = "{EYEBROW}"
```

### Step 5: Generate prompt.md

Use the template below, replacing all `{TOPIC_*}` variables with topic-specific content.

### Step 6: Confirm

Show the user:
- Path to generated `prompt.md`
- How to run it:
  ```bash
  claude --task "$(cat ~/newsletters/topics/{TOPIC_SLUG}/prompt.md)" \
    --channels plugin:telegram@claude-plugins-official \
    --dangerously-skip-permissions --max-turns 25
  ```
- The portal URL after build: `http://100.110.249.12:8787/{TOPIC_SLUG}/`

## Prompt Template

```markdown
# {TOPIC_NAME} — Automated Newsletter

Today's date: determine from system clock.
You are a technical newsletter curator for {TOPIC_AUDIENCE}.
Your job: search the web thoroughly, compile findings, generate an HTML page, rebuild the portal, and share the link via Telegram.

---

## SECTION 1: {TOPIC_SECTION1_TITLE} (past 24 hours)

Search {TOPIC_OFFICIAL_SOURCES} and the web for:

{TOPIC_SECTION1_CATEGORIES}

If nothing new in a sub-category, skip it. Don't fabricate updates.

---

## SECTION 2: GitHub Repos (past 7 days)

Search GitHub for trending, new, or significantly updated repositories.

{TOPIC_GITHUB_CATEGORIES}

For EACH repo include:
- **Name** with GitHub link
- **Stars** (current count)
- **1-line description** of what it does
- **Why it's interesting** (1 sentence)
- **Difficulty** tag: [easy] [medium] [advanced]
  - Easy = install and go
  - Medium = some config, env vars, or Docker needed
  - Advanced = complex setup, multiple services

Aim for 5-8 repos total. Prioritize repos with recent commits (last 7 days),
meaningful star counts (50+), and clear READMEs.

---

## SECTION 3: Community & Research (past 24 hours)

{TOPIC_COMMUNITY_SOURCES}

---

## OUTPUT FORMAT

Generate `~/newsletters/topics/{TOPIC_SLUG}/site/index.html` by reading the template and replacing placeholders with content.

**DO NOT modify the design, colors, fonts, CSS, or layout. All styling is locked.**

Reference files (read these before generating):
- **Template**: `~/newsletters/topics/{TOPIC_SLUG}/site/template.html`
- **Stylesheet**: `~/newsletters/topics/{TOPIC_SLUG}/site/style.css`

**Placeholders to replace:**

| Placeholder | Value |
|---|---|
| `{{DATE_TITLE}}` | e.g. `March 22, 2026` |
| `{{DATE_LONG}}` | e.g. `Saturday, March 22, 2026` |
| `{{SIGNAL_DOTS}}` | N filled + (5-N) empty signal dots |
| `{{SIGNAL_RATING}}` | e.g. `4 / 5` |
| `{{VERSION_RANGE}}` | Version/release range if applicable, or date range |
| `{{RELEASE_ITEMS}}` | One `.release-item` per release/update (version, date, description) |
| `{{HIGHLIGHT_CARDS}}` | One `.highlight-card` per highlight. Category classes as needed |
| `{{REPO_COUNT}}` | Number of repos |
| `{{REPO_CARDS}}` | One `<a class="repo-card">` per repo |
| `{{COMMUNITY_BLOCKS}}` | One `.community-block` per source |
| `{{TIP_CONTENT}}` | Tip text with code examples |
| `{{FOOTER_COLOPHON}}` | One-line day summary |

Each placeholder has example HTML in the template comments. Follow those patterns exactly.

If a section is empty, say "Nothing notable today" — don't pad with filler.

---

## ACTIONS (execute in order)

1. Search the web thoroughly for all sections above
2. Compile the newsletter content
3. Save raw content to: `~/newsletters/topics/{TOPIC_SLUG}/YYYY-MM-DD.md`
4. Generate the HTML page at: `~/newsletters/topics/{TOPIC_SLUG}/site/index.html`
5. Also save a dated copy at: `~/newsletters/topics/{TOPIC_SLUG}/site/YYYY-MM-DD.html`
6. Rebuild the portal: `cd ~/newsletters && uv run shared/build.py`
7. Ensure HTTP server is running on port 8787:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" http://localhost:8787/ | grep -q 200 || \
     (cd ~/newsletters/dist && python3 -m http.server 8787 &>/dev/null &)
   ```
8. Send one Telegram message (chat_id: 1538018072) with the Tailscale link: `http://100.110.249.12:8787/{TOPIC_SLUG}/`
9. Exit — do not wait for further input
```

## Highlight Card Categories

When generating highlight cards, define 4 category classes that fit the topic. Map them to the existing CSS color variables:

| CSS Class | Color | Use For |
|---|---|---|
| `voice` | Prussian blue | Product/UX updates |
| `model` | Terracotta | Core tech/engine updates |
| `company` | Sage green | Organization/team news |
| `promo` | Gold | Ecosystem/community/promos |

Document the chosen mapping in the generated prompt.md so future runs are consistent.

## Section 01 Adaptation Guide

Section 01 uses `.release-item` (version timeline). For topics without versioned releases, adapt:

| Topic Type | Section 01 Content |
|---|---|
| Software (Rust, K8s) | Version releases, changelogs, RFCs |
| Company/Product | Feature launches, pricing changes, partnerships |
| Research field | New papers, preprints, conference talks |
| Industry/Market | News, regulations, funding rounds |

The `.release-item` grid works for any dated entries — just use the date column and description.
