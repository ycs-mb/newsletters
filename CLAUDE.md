# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Automated newsletter portal. Three independent daily briefings (Claude Digest, US-Iran Conflict, Google AI) are generated by Claude Code via web search, compiled into HTML using a locked editorial design system, and served as a unified static site.

## Commands

```bash
# Build the portal (generates dist/ from topic HTML files)
uv run shared/build.py

# Run all newsletters + build + serve (full daily pipeline)
bash run.sh

# Serve the portal locally
cd dist && python3 -m http.server 8787

# Run a single newsletter (e.g., Google AI only)
claude --task "$(cat topics/google-ai/prompt.md)" \
  --channels plugin:telegram@claude-plugins-official \
  --dangerously-skip-permissions --max-turns 25
```

## Architecture

### Generation Flow

`topics/<slug>/prompt.md` → Claude Code (web search + compile) → `topics/<slug>/site/index.html` + `topics/<slug>/site/YYYY-MM-DD.html` → `shared/build.py` → `dist/`

Each topic has a `prompt.md` that instructs Claude Code to search the web, compile content, and generate HTML by filling `{{PLACEHOLDER}}` markers in the topic's `site/template.html`. The build script then assembles everything into `dist/` with navigation.

### Topic Structure

Topics are defined in `topics.toml`. Each entry maps a slug to an explicit folder under `topics/`, for example:

- `folder = "topics/claude-digest"`
- `folder = "topics/google-ai"`
- `folder = "topics/us-iran-war"`

Each topic folder contains: `prompt.md`, `site/template.html`, `site/index.html` (latest generated), `site/YYYY-MM-DD.html` (dated archives), and `YYYY-MM-DD.md` (raw markdown).

### Shared Infrastructure (`shared/`)

Shared infrastructure lives in `shared/`:

- `shared/build.py` — static site generator
- `shared/portal.css` — portal navigation and landing page styles
- `shared/templates/` — nav and landing page templates
- `shared/assets/style.css` — locked editorial design system copied into `dist/style.css`

### Build System (`shared/build.py`)

Pure stdlib Python (no deps, uses `tomllib`). Reads `topics.toml`, discovers dated HTML archives in each topic's `site/` folder, injects a navigation bar into each page, rewrites CSS paths, and generates a landing page. Output: `dist/` directory served on port 8787.

Key functions: `inject_nav()` inserts nav HTML after `<body>` tag via string replacement. `extract_metadata()` regex-parses signal ratings from generated HTML. `discover_html_archives()` finds `site/YYYY-MM-DD.html` files.

### Adding a New Topic

1. Create `topics/<slug>/` with `prompt.md` and `site/template.html`
2. Add any dated markdown and generated HTML under `topics/<slug>/`
3. Add entry to `topics.toml` with name, description, accent, folder
4. Run `uv run shared/build.py` — it discovers the new topic automatically

## Design System Rules

- **`shared/assets/style.css` is locked** — never modify it. All 769 lines are the editorial design system.
- **`shared/portal.css` supplements** — nav bar, landing page, archive styles. Uses CSS vars from `style.css`.
- **Templates use `{{PLACEHOLDER}}`** markers — generators fill them, never alter structure.
- **Accent colors**: `terracotta` (#c44b1a), `sage` (#2d6b42), `prussian` (#1a3552), `gold` (#9a7b1a)
- **Highlight card categories**: CSS classes `voice`, `model`, `company`, `promo` — same classes, semantic meaning varies per topic.
- **Typography**: Cormorant Garamond (display), Literata (body), Fira Code (mono)

## Serving

Single HTTP server on port **8787** serves `dist/`. Accessible via Tailscale at `http://100.110.249.12:8787`. Each topic at `/claude-digest/`, `/google-ai/`, `/us-iran-war/`.

## Telegram Integration

Newsletters send completion messages via the `plugin:telegram` channel. Chat ID `1538018072`. Links point to Tailscale URL + topic path.
